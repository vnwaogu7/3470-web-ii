Advance CSS Animations!

CSS3 paves the way to endless creations in html and CSS. The latest improvements to the language have allowed for more animations to be created in CSS. This can serve a useful tool for developers of all skill-levels, as it grants them the opportunity to create animations that were once only accessible with the use of Java.

The Break Down

In order to understand CSS animation, you must first understand its components. Advance CSS animations are created from @keyframes and Animation properties.

@keyframes + properties

@Keyframes:

A @keyframe is a CSS rule that specifies the animation of the code. It sets up the sequence for each animation.

animation.duration

-indicates how long the animation should last

animation.timing-function:

specifies the type of timing the animation will use to transition

linear

The animation has the same speed from start to end

ease

The animation has a slow start, then fast, before it ends slowly. This is a default.

ease-in

The animation has a slow start

ease-out

The animation has a slow end

ease-in-out

The animation has both a slow start and a slow end

animation.delay:

How long it takes for an animation to start

No Delay

2 Second Delay

animation.iteration-count:

-Represents the number of times the animation loops

Number (2,3,5, 6 etc.)

The number defines how many times the animation loops. In this example, the animation loops 3 times.

Infinite

The animations loops an infinite amount of times

animation.direction

The animation. direction indicates which direction the animation will be moving

normal

The animation is played as normal in the forward direction. This is also the default value of the animation

reverse

The animation is played in reverse direction.

alternate

The animation is played forwards first, then backwards.

alternate-Reverse

The animations loops an infinite amount of times

animation.fill-mode

animation.fill-mode specifies a style for the element when the animation is not playing (before it starts, after it ends, or both)

animation.play-state
